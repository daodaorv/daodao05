### 3. 业务概念错误

#### 问题7: 合作管理模块概念错误 ⚠️ 严重
**问题描述**:
- 当前设计将"合作商"和"供应商"混为一谈
- 合作商车辆未使用统一车型库
- 假设可以通过API自动对接合作商系统（实际需要手动对接）

**正确设计**:
- **合作商**: 房车租赁供应商（房车生活家等），提供房车租赁服务
- **供应商**: 服务供应商（维修厂、保险公司等），提供维保/保险服务
- 合作商门店在"门店管理"中统一管理
- 合作商车辆使用统一车型库，绑定到合作商门店
- 用户下单后，管理员手动到合作商网站下单，填入订单信息
- 系统自动计算差价和门店分润（默认30%）

**影响范围**:
- 产品需求文档（第3.6节）
- 实施计划文档（步骤2.5、2.6）
- 菜单架构设计
- 数据库设计

**修复优先级**: P0（关键）

**修复建议**:
1. 修正产品需求文档，明确区分合作商和供应商
2. 将合作商管理和供应商管理拆分为独立模块
3. 在门店管理中增加"合作商门店"类型
4. 在车辆管理中增加"合作商车辆"来源
5. 在订单详情中增加"合作商订单信息"区域
6. 在分润配置中增加"合作商订单门店分润比例"配置
